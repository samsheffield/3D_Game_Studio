INTRODUCTION TO VERSION CONTROL & UNITY

WHY VERSION CONTROL?

  • File management is messy
  • It provides a great backup & archiving solution for text-based work
  • It can help make collaborative projects easier to manage

INTRO TO GITHUB

  • Register an account with GitHub (if you don't have one)
  • Download & install the GitHub Desktop app
  • Launch app and connect to your account. Configure with your name and email address
  • Creating & cloning repositories
  • .gitignore file
  • Creating commits
  • Reverting changes
  • Working in branches
  • Merging branches

UNITY SPECIFIC SETTINGS FOR VERSION CONTROL

  1. Change Project Settings > Editor > Version Control: Visible Meta Files
  2. Change Project Settings > Editor > Asset Serialization: Force Text
  3. File > Save Project

GOOD PRACTICES

  1. No source material for assets in the repository, only exported assets (Continue to use Drive/Dropbox to share, or backup, these things.
  2. Always use a .gitignore file. There are a number of files which do not need to be uploaded & synced with teammates
  3. One person creates the repository and adds others as collaborators through its settings (gear icon in web interface)
  4. Use prefabs as much as possible when working in groups
  5. Always work in branches and merge changes
  6. In groups, delegate one person to control the merging of the master branch
